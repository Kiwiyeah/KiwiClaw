# 运维SOP（管理员级别）

## 日常检查
- 状态：`openclaw-cn gateway status`
- 健康检查：`openclaw-cn health`
- 控制台：`http://127.0.0.1:18789/`

## 故障处理
### 1. 健康检查失败
- 确认网关进程是否运行
- 查看日志并定位错误原因
- 必要时执行 `openclaw-cn gateway restart`

### 2. 端口冲突
- 确认端口占用与冲突进程
- 调整端口或停止冲突进程

### 3. 认证异常
- 重新确认 token 是否已注入服务环境变量
- 控制台重新输入 token

## 应急策略
- 高风险时临时停用敏感通道或插件
- 旋转 token 与 API key
- 记录并归档变更与影响范围
